Java面向对象的三大特征：继承封装 多态 封装说明一个类行为和属性与其他类的关系低耦合和高内聚， 继承是父类和子类的关系，多态是类和类的关系：父类引用指向子类对象、继承、重写。

增加了代码的复用性 可移植性健壮性

引用类型有哪些：

强引用类型：Byte a = new Byte(),代表了强引用对象。

强引用对象的生命周期：
当垃圾回收的时候，gc调用finalize()方法。

没有引用指向对象时候，对象就可以被回收M 吗 = new M(); m = null;

System.in.read() 阻塞主线程。

软引用 特殊的强引用

SoftRefence<byte[]> sr = new SoftRefence<>(new byte[1024 * 1024]);

 sr.get() 获得软引用。

当内存不够用的时候，使用gc进行回收，但是要是还不够用，则软引用不需要使用了。

软引用非常适合在缓存中使用。

弱引用

WeakRefence wr = new WeakRefences(new M)

弱引用只要是垃圾回收进行回收，则直接回收。

内存溢出，大对象不够内容存储。内存泄漏：大对象不能被回收，代表的是内存泄漏。

ThreadLocal 线程私有的一个容器。

ThreadLocal.set() ->Thred.threadLocals: Map对象 然后set（ThreadLocal对象和value）

ThreadLocal不使用的话，需要调用 tl.remove(); 解决ThreadLocal的value的引用。解决value的内存泄漏的问题

ThreadLocal不使用的话，将ThreadLocal tl = null,则当前线程的map对象中的key（即是ThreadLocal）仅有一条map中的虚引用指向它，解决key的内存泄漏的问题

线程池的代码：在线程使用过程中，需要在线程使用的需要把ThreadLocal的引用全部remove() 或者threadLocal.set(null)

应用：当方法调用参数传递的时候，ThreadLocal相当于一个容器，能够保存数据。让同一个线程中的参数进行调用。

spring中的@Tration事务的应用：中的connection的应该都放在THreadLocal中。(分布式事务)

堆外内存：jvm能够使用的以外的内存，不属于垃圾回收器管理的内存。

ByteBuffer buffer = ByteBuffer.allocateDirect( q)

DirectByteBuffer >> zero copy

拥有堆外内存的对象需要向GC通知管理堆外内存。

事务的特点：ACID

原子性由undoload日志保证的，记录了需要回滚的日志信息，事务回滚时候撤销已经执行的操作

一致性：由其他三大特性保证一致性

隔离性：由于MVCC来保证 多版本并发控制

持久性：由于redolog保证，redolog记录一份日志数据，就算没有修改成功只要日志保存成功了，数据仍然不会丢失。
